继承问题

调用子类构造方法时必须先调用父类的构造方法

如果父类没有无参构造那么子类中必须显式的调用父类的构造方法super()

注意：当一个类实现了有参构造，那么其默认将不会实现无参构造

所以此处会编译错误

实现或继承了Collection接口的有

在java.util包中提供了一些集合类，

常用的有List、Set和Map类，其中List类和Set类继承了Collection接口。

这些集合类又称为容器，长度是可变的，数组用来存放基本数据类型的数据，集合用来存放类对象的引用。
List接口、Set接口、Map接口以及Collection接口的主要特征如下：

  • Collection接口是List接口和Set接口的父接口，通常情况下不被直接使用。
  • List接口继承了Collection接口，List接口允许存放重复的对象，排序方式为按照对象的插入顺序。
  • Set接口继承了Collection接口，Set接口不允许存放重复的对象，排序方式为按照自身内部的排序规则。
  • Map接口以键值对（key—value）的形式存放对象，其中键（key）对象不可以重复，值（value）对象可以重复，排序方式为按照自身内部的规则，没有继承或实现Collection接口。

Vector实现了List接口，即间接实现Collection接口

Iterator是Java迭代器最简单的实现，没有继承或实现Collection接口。

JVM，JDK，JRE

JVM：即Java虚拟机，运行Java字节码文件，时Java跨平台的关键

JDK（Java Development Kit）：包含JRE所拥有的一切，除此之外还有编译器（javac）和工具（javadoc,jdb），它能创建和编译程序

JRE：Java运行时环境，它是运行已编译Java程序所需所有内容的集合，包括JVM，Java类库，Java命令和一些其他构件。但其无法用于创建Java程序。

如果你只是为了运行一下 Java 程序的话，那么你只需要安装 JRE 就可以了。如果你需要进行一些 Java 编程方面的工作，那么你就需要安装 JDK 了。但是，这不是绝对的。有时，即使您不打算在计算机上进行任何 Java 开发，仍然需要安装 JDK。例如，如果要使用 JSP 部署 Web 应用程序，那么从技术上讲，您只是在应用程序服务器中运行 Java 程序。那你为什么需要 JDK 呢？因为应用程序服务器会将 JSP 转换为 Java servlet，并且需要使用 JDK 来编译 servlet。

什么是字节码？，字节码的好处

字节码，即能被JVM理解的代码就叫做字节码（.class文件）不面向任何特定的处理器，只面向JVM

字节码文件是经过编译器预处理过的一种文件,

是Java的执行文件形式,本身是二进制文件,但是不可以被系统直接执行,

而是要虚拟机解释执行,由于被预处理过,所以比一般的解释代码要快,但是仍然比系统直接执行的慢

在一定程度上解决了传统解释型语言执行效率低的问题，同时又保留了解释型语言可移植的特点。

Java程序的执行流程

没有JIT之前，JVM加载字节码文件，然后通过解释器逐行解释执行，执行速度较慢，

引进JIT后，由于JIT属于运行时编译，所以当JIT完成一次编译后，会将字节码对应的机器码保存下来以便下次直接使用

而我们知道，机器码的运行效率肯定是高于 Java 解释器的。这也解释了我们为什么经常会说 Java 是编译与解释共存的语言

HotSpot 采用了惰性评估(Lazy Evaluation)的做法，根据二八定律，消耗大部分系统资源的只有那一小部分的代码（热点代码），而这也就是 JIT 所需要编译的部分。JVM 会根据代码每次被执行的情况收集信息并相应地做出一些优化，因此执行的次数越多，它的速度就越快。JDK 9 引入了一种新的编译模式 AOT(Ahead of Time Compilation)，它是直接将字节码编译成机器码，这样就避免了 JIT 预热等各方面的开销。JDK 支持分层编译和 AOT 协作使用

JVM

Java中只有栈帧没有堆帧，PC寄存器就是程序计数器

程序计数器是一个以线程私有的一块较小的内存空间，用于记录所属线程所执行的字节码的行号指示器；字节码解释器工作时，通过改变程序计数器的值来选取下一条需要执行的字节码指令，分支、循环、跳准、异常处理、线程恢复等基础功能都需要依赖程序计数器来完成

HashMap与Hashtable的区别

  • 这两货继承的类不同，但是都实现了map接口。
  • 从线程安全的角度来看：
  • Hashtable是线程安全的，它的每个方法中都加入了Synchronize方法。在多线程并发的环境下，可以直接使用Hashtable，不需要自己为它的方法实现同步
  • HashMap不是线程安全的，在多线程并发的环境下，可能会产生死锁等问题。具体的原因在下一篇文章中会详细进行分析。使用HashMap时就必须要自己增加同步处理，
  • 虽然HashMap不是线程安全的，但是它的效率会比Hashtable要好很多。这样设计是合理的。在我们的日常使用当中，大部分时间是单线程操作的。HashMap把这部分操作解放出来了。当需要多线程操作的时候可以使用线程安全的ConcurrentHashMap。ConcurrentHashMap虽然也是线程安全的，但是它的效率比Hashtable要高好多倍。因为ConcurrentHashMap使用了分段锁，并不对整个数据进行锁定。

Java的三大特性

封装

把同一类事物对的共性抽离成为一个类，

可以隐藏类的内部细节，用户无需了解其内部实现，只需调用对象提供的外部接口访问此类

实现专业的分工，程序员可以按需调用

继承

子类继承父类，子类可以拥有弗雷除私有属性，方法和构造方法外的所有的属性和方法

继承堆修改关闭，对扩展开放，遵循里氏代换原则，子类可以写自己特有的属性和方法，也可以复写弗雷的方法，这就叫方法重写

继承可以实现代码的复用

多态

对象的多态性，父类的引用指向子类的对象（或子类的对象赋给父类的引用）

多态存在的三个必要条件

  1. 继承

  2. 重写

  3. 父类引用指向子类对象

    相同的事物，使用相同的参数调用相同的方法，结果却不相同。

    子类以父类的身份出现时，需要向上转型，这是JVM自动实现的，是安全的，

    但向下转型是不安全的，需要强制转换，

    子类以父类身份出现时自己特有的属性和方法不能使用

Java的基本数据类型以及包装类

基本类型 大小（字节） 默认值 包装类
byte 1 (byte)0 Byte
short 2 (short)0 Short
int 4 0 Integer
long 8 0L Long
float 4 0.0f Float
double 8 0.0d Double
boolean - false Boolean
char 2 null(\u0000) Character
  • boolean在数组中只占1个字节，不在数组中是占4个字节
  • 基本数据类型所占用的空间是不变的，这也是Java具有可移植性的原因之一
  • 基本数据类型放在栈中，直接 存储

为什么需要包装类

  • 基本类型不是类，不能new出来，因此不具备面对对象的功能，无法调用方法。
  • 在一个类或接口或方法中定义一个泛型的数据类型，当使用这个类、接口、方法时，要把泛型定义成具体的基本数据类型就必须使用基本数据类型对应的包装类进行定义。

包装类与基本数据类型的不同

  • 基本类型只能按值传递，而封装类型按引用传递
  • 基本类型在栈中创建，而对于对象类型，对象在堆中创建，对象的引用在栈中创建，基本类型由于在栈中，执行效率会比较高，但可能存在内存泄漏的问题。

  1. jdk8以及之后的版本，HashMap的数据结构为数组+链表/红黑树
  • 准确的说HashMap的底层是哈希表，而Java中哈希表是由数组+链表实现的，每一个数组的节点称之为bucket桶，当桶中由多个元素就以链表的方法进行存储，当添加元素时，通过key的hash算法得出hash值，然后通过hash值映射出存在哪个桶中，如果桶中没有有元素，就添加成功，如果有元素那就代表散列冲突，比较已存在的元素和新插入的元素的hash值是否有相同，如果没有相同的则添加成功(情况1)，如果某个元素的hash值和新的元素hash值相同的，则调用key的equals方法继续进行比较，为true则新的值覆盖掉旧的值，为false添加成功(情况2)，对于情况1 和情况2，在jdk1.7是将新的元素添加到原来的头节点，jdk1.8是添加到链表后面，jdk1.8做了一个优化，因为如果链表的长度太长遍历查找复杂度为O(n)，所以引入了红黑树，当桶中的元素>8并且数组长度>64就转为红黑树，复杂度为O(logn)
  1. C语言中每个字符串末尾都有一个 \0 结束 符号，所以字符串占用的字节数=顺序存储字符串的长度+1
  2. 查询数组中第i个元素的平均时间复杂度为O(1)

线性与顺序结构

线性是线性，顺序是顺序，线性是逻辑结构，顺序是存储结构，两者不是一个概念，线性是指一个元素后继只有唯一的一个元素或节点，非线性是一个元素后面可以有多个后继或前继节点，顺序是指存储结构连续，例如数组是顺序的，链表不是顺序的，但他们都是线性的。当然顺序也可以是非线性的，例如顺序结构存储非线性结构的二叉树！！！

线性结构指的是逻辑结构，而不是物理存储结构
常用的线性结构有：线性表，堆栈，队列，双队列，数组，串。
常见的非线性结构有：二维数组，***数组，广义表，树(二叉树等)，图。
